# 💣 BOMB DEFUSAL — 1:1 協力型 爆弾解除ゲーム

2人のプレイヤーが WebSocket（TCP）でリアルタイム通信しながら協力して爆弾を解除するゲーム。
プレイヤーA（爆弾担当）は **3Dの爆弾** のみ、プレイヤーB（マニュアル担当）は **解除マニュアル** のみが見える非対称協力ゲーム。
「Keep Talking and Nobody Explodes」風。制限時間内に4つのモジュールをすべて解除すればクリア、ミス3回または時間切れで起爆。

## 通信方式

- **プロトコル:** WebSocket（TCP ベース）
- **通信形態:** 1:1（A ↔ サーバー ↔ B）
- **メッセージ形式:** JSON

## 起動方法

```bash
# 依存パッケージのインストール
npm install

# サーバー起動
npm start
# → http://localhost:3000 にアクセス
```

### ローカルで2台の端末を接続する場合

1台目（サーバーを起動するPC）と2台目（スマホ・別PCなど）を同じWi-Fiに接続する。

```bash
npm start
```

起動後、ターミナルに次のようなURLが表示される:

```text
同じWi-Fiの別端末からアクセス:
  http://192.168.x.x:3000
```

- 1台目: `http://localhost:3000` または表示された `http://192.168.x.x:3000` を開く
- 2台目: `http://192.168.x.x:3000` を開く
- 2台目で `localhost` を開くと、2台目自身を見に行くため接続できない
- 接続できない場合は、Mac/Windows のファイアウォールで Node.js の受信接続を許可する

2つのブラウザタブ（または2台の端末）で開く:
1. 1人目: 「ルームを作成する」→ 表示された6桁コードを相手に伝える
2. 2人目: コードを入力して「参加」
3. 両者がロール（A/B）を選択するとゲーム開始

> **注意:** 爆弾側の3D表示は Three.js を CDN から読み込むため、プレイには **インターネット接続** が必要です。

### 別の端末と遊ぶ
- **同一LAN内:** サーバー機のIPアドレス（`ipconfig` で確認）に `http://<IP>:3000` でアクセス。Windows ファイアウォールで3000番の許可が必要。
- **大学Wi-Fi等（端末分離あり）/ 別ネットワーク:** `ngrok http 3000` でトンネルを張り、表示された `https://xxxx.ngrok-free.dev` を相手に渡す。

## ゲーム内容（4モジュール）

| モジュール | 爆弾側の操作 | マニュアル側で読むもの |
|-----------|------------|----------------------|
| **① WIRES（ワイヤー）** | 絡まった配線から1本をクリックして切断 | 本数・色・シリアル奇偶による切断条件表 |
| **② KEYPAD（コード）** | 4桁を規則変換して入力 | 合計の一の位K→各桁+K→シリアルで並べ替え |
| **③ MORSE（モールス）** | LED点滅を再生し、周波数CHを選んで送信 | 記号→モールス→CH番号の対応表 |
| **④ SIMON（記憶）** | 点滅色を覚え、変換した色を順に押す | シリアル奇偶で切り替わる色変換表 |

- 爆弾側UIは **Three.js による3D**。1つの筐体に4ギミックが埋め込まれ、固定正面視点でクリック操作する。
- マニュアル側は **白紙・黒インクのタイプライター体**（あえて色を付けず読みにくくして難度を上げている）。
- 答えはサーバー内部にのみ保持し、爆弾側クライアントには送らない（`sanitizeForClient`）。

## 主なメッセージ（JSON）

| type | 方向 | 用途 |
|------|------|------|
| `room:create` / `room:join` | C→S | ルーム作成・参加 |
| `role:select` | C→S | ロール（A/B）選択 |
| `game:start` | S→C | ゲーム開始（役割・パズルデータ・状態） |
| `game:state` | S→C | 毎秒の残り時間同期 |
| `puzzle:wire:cut` / `puzzle:code:submit` / `puzzle:morse:submit` / `puzzle:simon:submit` | C→S | 各モジュールの解答 |
| `puzzle:result` | S→C | 正誤・ミス数・解除状況 |
| `game:clear` / `game:over` | S→C | クリア / 起爆 |
